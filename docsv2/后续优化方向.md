## 一、Workflow 编排

### 是什么

Workflow 是真实 CC 的多 Agent 编排引擎。核心概念：

```
用户任务
  → Workflow 脚本（JavaScript）
    → pipeline(items, stage1, stage2, ...)  ← 流水线，无 barrier
    → parallel(thunks)                       ← 并行，有 barrier
    → agent(prompt, opts)                    ← 启动子 Agent
    → phase(title)                           ← 进度分组
  → 多个子 Agent 按拓扑执行
  → 结果汇总、去重、验证、合成
```

### 核心原语

| 原语 | 行为 |
|------|------|
| `pipeline(items, stage1, stage2)` | 每个 item 流经所有 stage，item 之间无 barrier |
| `parallel(thunks)` | 所有 thunk 并发执行，等待全部完成（barrier） |
| `agent(prompt, {schema, model, isolation})` | spawn 一个子 Agent，可返回结构化数据 |
| `phase(title)` | 进度分组标签 |

### 为什么不是 P0/P1

Workflow 会放大当前系统的不足：
- 工具选择混乱 → 多个 Agent 各自选错工具
- 文件写入冲突 → 并行修改同一个文件
- 权限不完整 → 子 Agent 绕过权限
- 验证闭环不稳定 → 多个 Agent 各自失败

必须等单 Agent 闭环完全稳定后再做。

### 后端设计

**新增模块**：

| 模块 | 职责 |
|------|------|
| `src/main/agent/WorkflowEngine.ts` | 解析 Workflow 脚本，调度 pipeline/parallel/agent |
| `src/main/agent/WorkflowScriptParser.ts` | 安全沙箱执行用户 JS 脚本 |
| `src/main/agent/AgentBlackboard.ts` | 跨 Agent 共享上下文（API 契约、关键发现、文件边界） |

**Workflow 工具**：
```ts
Workflow({
  script: `
    export const meta = { name: 'review-changes', ... }
    const results = await pipeline(DIMENSIONS,
      d => agent(d.prompt, {schema: FINDINGS_SCHEMA}),
      review => parallel(review.findings.map(f => 
        () => agent("Verify: " + f.title, {schema: VERDICT_SCHEMA})
      ))
    )
  `
})
```

**安全性**：
- 脚本在受限沙箱中执行（无 fs/net 访问）
- `Date.now()` / `Math.random()` 被禁止（保证幂等 resume）
- 最大 1000 agent 调用上限
- 并发上限 min(16, cpuCores - 2)

### 前端

| 组件 | 职责 |
|------|------|
| `WorkflowProgressPanel.tsx` | 多轨道 Agent 进度展示 |
| `AgentTrack.tsx` | 单个 Agent 的独立进度条 |
| `BlackboardView.tsx` | 共享上下文可视化 |

**工作量**：大（~2000 行，Workflow 引擎 + 脚本解析 + Blackboard + 3 前端组件 + 安全沙箱）

## 二、MCP 接入

### 是什么

MCP（Model Context Protocol）是把外部系统接入 Agent 的标准协议。一个 MCP Server 可以提供：

| 能力 | 示例 |
|------|------|
| **tools** | 查询数据库、控制浏览器、读 GitHub issue |
| **resources** | 文档 schema、API 描述、设计稿 |
| **prompts** | 可复用提示模板 |

### 为什么不是 P0/P1

MCP Server 接入后，Agent 会获得更多工具。但如果 Agent 本身的基础执行能力不稳固：
- 外部工具会分散模型注意力
- 增加权限管理复杂度
- 错误更难排查

### MCP 调用流程

```
Agent 请求调用 MCP 工具
  → Runtime 检查权限和参数
  → MCP Client 发送请求到 MCP Server（通过 stdio/SSE/WebSocket）
  → MCP Server 访问外部系统（数据库、浏览器、GitHub API）
  → 返回标准化响应
  → Agent 继续推理
```

### 后端设计

**新增模块**：

| 模块 | 职责 |
|------|------|
| `src/main/mcp/McpClient.ts` | MCP 协议客户端（stdio/SSE 传输） |
| `src/main/mcp/McpServerManager.ts` | 管理已注册的 MCP Server 生命周期 |
| `src/main/mcp/McpToolProxy.ts` | 把 MCP tools 代理为 CodeZ Tool 接口 |
| `src/main/tools/builtin/McpTool.ts` | 暴露给模型的统一 MCP 调用工具 |

**MCP Server 配置**（`~/.codez/mcp-servers.json`）：

```json
{
  "servers": [
    {
      "name": "browser",
      "command": "node",
      "args": ["mcp-servers/browser/server.js"],
      "env": {}
    },
    {
      "name": "github",
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-github"],
      "env": { "GITHUB_TOKEN": "${env:GITHUB_TOKEN}" }
    }
  ]
}
```

### 前端

| 组件 | 职责 |
|------|------|
| `McpSettingsTab.tsx` | MCP Server 管理界面（安装/启用/禁用） |
| `McpServerStatus.tsx` | 每个 Server 的连接状态指示灯 |

### 权限

MCP 调用走同一套权限系统，不能绕过。

**工作量**：大（~1500 行，McpClient + ServerManager + ToolProxy + 配置系统 + 2 前端组件）

## 三、Swarm 多 Agent 并发

### 是什么

Swarm 是比 Workflow 更高层的多 Agent 协作框架。角色分工：

```
User
  → Manager Agent (需求理解、任务拆解、DAG、结果合并)
    → Scout Agent ×N (并行搜索、读取、总结上下文)
    → Coder Agent ×N (并行实现独立模块)
    → QA Agent (运行测试、验证、诊断失败)
  → AgentBlackboard (共享关键上下文)
  → Final Report
```

### 角色权限

| 角色 | 职责 | 工具权限 |
|------|------|---------|
| Manager | 需求理解、任务拆解、DAG、合并 | search/read only，不写文件 |
| Scout | 快速搜索、读取、总结 | search/read only |
| Coder | 修改授权范围内文件 | search/read/Edit/Write、有限 shell |
| QA | 测试、验证、诊断 | read/shell，默认不写文件 |

### 核心模块

| 模块 | 职责 |
|------|------|
| `RoleConfig` | 定义每种角色的 system prompt、工具列表、fileScope、并发数 |
| `SwarmDispatcher` | 接收 DAG → 拓扑排序 → 控制并发 → 启动 AgentRunner → 收集结果 |
| `AgentBlackboard` | 保存 API 契约、关键发现、文件责任边界、GoalSnapshot、DecisionLog |

### 冲突检测

- 同一文件不能被多个 Coder 同时写入
- DAG 有环时校验失败
- 任一子 Agent 失败不破坏全局

### 前端

| 组件 | 职责 |
|------|------|
| `SwarmDashboard.tsx` | 多 Agent 进度可视化 |
| `RoleLegend.tsx` | 颜色编码的角色图例 |
| `BlackboardPanel.tsx` | 共享上下文浏览器 |

**工作量**：大（~3000 行，SwarmDispatcher + RoleConfig + Blackboard + SwarmDashboard + 冲突检测 + DAG 校验）
