# Plan 模式实现计划

> 设计文档：`docs/superpowers/specs/2026-07-01-plan-mode-design.md`
> 状态：approved → 实施中
> 范围：前端 Toggle + Plan 胶囊 + 后端只读约束 + System Prompt

## 0. 关键纠正 — 工具名映射

设计文档使用 Claude Code 风格名称，**实际代码工具名**如下（必须使用这些）：

| 设计文档名 | 实际工具名 |
|-----------|-----------|
| `read_file` | `Read` |
| `list_files` | `list_files` |
| `glob` | `Glob` |
| `grep` | `Grep` |
| `get_project_snapshot` | `get_project_snapshot` |
| `fast_context` | `fast_context` |

只读工具名集合：`Read`, `list_files`, `Glob`, `Grep`, `get_project_snapshot`, `fast_context`

## 1. 数据流总览

```
PromptArea[Toggle]
  → chatStore.planMode (zustand)
    → useSendMessage 读取 planMode
      → window.api.chat.stream(..., { planMode })  [preload]
        → IPC CHAT_STREAM_START 携带 planMode
          → chat.handlers StreamRequest
            → AgentRunner.run({ planMode })
              ├─ planMode=true → getReadOnlyTools()
              ├─ 注入 Plan 模式引导消息
              └─ 发送 PLAN_STATE_CHANGED 到前端
```

## 2. 任务拆解

### Phase 1：后端 — ToolManager + AgentRunner + handlers + SystemPrompt

**T1.1 ToolManager.getReadOnlyTools()**
- 文件：`src/main/tools/ToolManager.ts`
- 新增静态集合 `READ_ONLY_TOOL_NAMES = new Set(['Read', 'list_files', 'Glob', 'Grep', 'get_project_snapshot', 'fast_context'])`
- 新增方法 `getReadOnlyTools(): ToolDefinition[]` 过滤 `getAllTools()`
- 注意：`getAllTools()` 返回 `Tool[]`，映射到 `ToolDefinition` 格式与 `getToolDefinitions()` 一致

**T1.2 AgentRunner planMode 支持**
- 文件：`src/main/agent/AgentRunner.ts`
- `AgentRunConfig` 接口新增 `planMode?: boolean`（已在 ChatRequestConfig 通过 extend 加字段）
- `run()` 方法内：
  - `const availableTools = config.planMode ? this.toolManager.getReadOnlyTools() : (config.tools || this.toolManager.getToolDefinitions())`
  - 当 `config.planMode` 时，在 `allMessages` 的 system 消息后注入 Plan 模式引导消息（role: 'system'）
  - 通过 `mainWindow.webContents.send(IPC_CHANNELS.PLAN_STATE_CHANGED, { state: 'active', mode: config.planMode ? 'plan' : 'normal' })` 通知前端
  - 需要获取 `BrowserWindow` 引用：通过 `import { BrowserWindow } from 'electron'` 和当前 webContents

**T1.3 chat.handlers planMode 透传**
- 文件：`src/main/ipc/chat.handlers.ts`
- `StreamRequest` 接口新增 `planMode?: boolean`
- `runner.run()` 调用时传入 `planMode: request.planMode`

**T1.4 SystemPromptService Plan 模式段落**
- 文件：`src/main/services/SystemPromptService.ts`
- 在 `buildDeveloperInstructions()` 的 `</developer_instructions>` 前新增 PLAN MODE 段落

### Phase 2：前端 — chatStore + preload + useSendMessage

**T2.1 chatStore planMode 状态**
- 文件：`src/renderer/src/stores/chatStore.ts`
- `ChatState` 新增 `planMode: boolean`（默认 false）
- 新增 action `setPlanMode: (mode: boolean) => void`
- 新增 action `togglePlanMode: () => void`（方便 Toggle 按钮调用）
- 初始化 `planMode: false`
- 新增 `initPlanStateListener()` 监听 `PLAN_STATE_CHANGED`，更新 planMode（Agent 完成后设为 false）

**T2.2 preload chat.stream 透传 planMode**
- 文件：`src/preload/index.ts`
- `chat.stream` 签名新增第 5 参数 `planMode?: boolean`
- 调用 IPC 时传入 `{ providerId, model, messages, sessionId, planMode }`

**T2.3 useSendMessage 读取并传递 planMode**
- 文件：`src/renderer/src/components/chat/hooks/useSendMessage.ts`
- 从 store 读取 `planMode`
- 调用 `window.api.chat.stream()` 时传入 `planMode`

### Phase 3：前端 — UI 组件

**T3.1 PromptArea Plan Toggle 按钮**
- 文件：`src/renderer/src/components/PromptArea.tsx`
- 在输入框左侧（`prompt-top-toolbar` 或 `prompt-actions-left`）添加 Toggle 按钮
- 按钮样式：OFF 灰色轮廓 `[○ Plan]`，ON 橙色填充 `[● Plan]`
- 点击调用 `togglePlanMode()`
- Tooltip: "Plan mode: read-only exploration and design"

**T3.2 PromptArea.css Toggle 样式**
- 文件：`src/renderer/src/components/PromptArea.css`
- 新增 `.plan-toggle-btn` / `.plan-toggle-btn--active` 样式（橙色主题）

**T3.3 PlanPanel 胶囊组件**
- 新文件：`src/renderer/src/components/chat/PlanPanel.tsx`
- 新文件：`src/renderer/src/components/chat/PlanPanel.css`
- 参考 TaskPanel 模式：收缩态 capsule + 展开态 panel
- 收缩态：`📋 Plan` 橙色胶囊，仅在 `planMode=true` 时显示
- 展开态：显示允许的工具列表 + 状态文本 + "Recent plan" 步骤检测（正则匹配 `数字. 文本` 或 `- 文本`）

**T3.4 ChatAreaLayout 集成 PlanPanel**
- 文件：`src/renderer/src/components/chat/ChatAreaLayout.tsx`
- Plan 胶囊与 Task 胶囊互斥：`planMode=true` 时显示 PlanPanel，隐藏 TaskPanel；`planMode=false` 时显示 TaskPanel
- 注意：`expandedCapsule` 已支持 `'plan'` 值

### Phase 4：测试
