# Read 首次范围策略设计

## 目标

让模型首次读取普通已知文件且没有证据支持的相关范围时省略 `offset` 和 `limit`，由 `Read` 的现有内容预算返回文件；仅在范围已经有依据时使用按行读取，避免任意读取前 50/100 行后再发起下一轮调用。

## 方案选择

- **采用：提示词约束。** 保留 `offset`/`limit` 参数和运行时行为，只修改工具描述、系统规则及回归测试。
- **不采用：删除范围参数。** 这会破坏大文件精确读取和上下文裁剪后的恢复能力。
- **不采用：运行时自动忽略首次范围。** 工具无法可靠判断模型是否已通过搜索得到精确行号。

## 规则

- 首次读取且没有证据支持的相关范围时，默认只传 `file_path`。
- Grep 或其他已有证据已经定位相关范围时，即使是首次读取也可以传入 `offset`/`limit`。
- 不得用任意的前 50/100 行进行试探性浏览。
- 只有以下情况可以显式传入 `offset`/`limit`：
  - Grep 或其他已有证据给出了相关行范围；
  - 默认读取结果明确标记为截断，或已经到达工具注明的内容预算边界；
  - 上下文裁剪移除了之前已经读取的内容，需要恢复特定范围。
- 文件发生变化允许重读，但本身不构成随意限制到前若干行的理由。
- 批量读取、范围合并、溢出批次同轮并行等现有规则保持不变。

## 非目标

- 不修改 `Read` schema 或执行逻辑。
- 不删除 `offset`/`limit`。
- 不调整现有行数、字符数或文件数预算。
- 不修改已有历史设计文档。

## 验证

- `ReadTool.description` 明确要求没有相关范围依据的首次读取省略范围参数，并列出三个例外及现有 1,200 行默认边界。
- 生成的系统提示包含相同决策规则。
- 定向测试、全量测试、类型检查和生产构建通过。
