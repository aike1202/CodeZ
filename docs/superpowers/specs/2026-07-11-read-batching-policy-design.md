# Read 批量策略优化设计

## 目标

让模型在已知多个文件或行范围时优先使用一次 `Read.files` 批量读取，减少连续单项读取造成的不必要 Agent Loop，同时保留真正依赖前次结果的渐进式调查能力。

## 工具描述

`ReadTool.description` 增加以下决策规则：

- 调用前先收集当前已知的所有独立文件和范围。
- 已知两个或更多目标时，使用参数结构允许的最少 `files` 数组数量。
- 目标超过单个数组容量时，在同一次模型输出中并行发出额外的独立 `Read` 调用，不新增或硬编码容量限制。
- 同一文件的相邻或重叠范围应先合并为一个更大范围。
- 只有确实只有一个目标，或下一个目标依赖本次返回内容时，才使用单项数组。
- 不得为了逐段浏览而把已经知道的范围拆成多个连续 `Read` 调用。
- 仅在文件发生变化、之前内容已被上下文裁剪或新范围由当前结果揭示时重读。

## 全局执行规则

### Harness

在并行工具规则旁增加批量读取优先级：已知的独立读取目标应使用参数结构允许的最少 `Read.files` 调用；超出单次容量的批次在同一次模型输出中并行发出，而不是跨多个模型轮次逐项调用。

### ToolPolicy

将“Read a file”改为“Read one or more known files or ranges”，并明确单次 `Read` 的 `files` 数组是默认选择。只有数据依赖存在时才分开调用。

### Investigation

- 在开始读取前先通过 Glob/Grep 收集目标文件和范围。
- 将目标文件、直接邻居、调用方、被调用方和测试中已经知道的独立读取合并。
- 把误导性的 “Read twice, edit once” 改成 “Plan reads, batch known targets, then edit.”
- 明确渐进读取只适用于后续目标无法在当前结果前确定的情况。

## 非目标

- 不增加运行时重叠范围硬拦截。
- 不修改 `ReadTool` 参数结构或读取实现。
- 不调整单文件输出限制、指纹或上下文裁剪机制。
- 不修改历史设计文档。

## 验证

- `ReadTool.description` 包含批量、合并范围和依赖例外规则。
- 生成的系统提示包含批量读取优先规则。
- ToolPolicy 不再使用单数 “Read a file”。
- Investigation 不再包含 “Read twice, edit once”。
- 全量测试、类型检查和生产构建通过。
