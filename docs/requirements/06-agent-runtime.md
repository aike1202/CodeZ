# 06. Agent Runtime 与任务执行

> 模块：Agent Loop、任务模式、计划展示、流式响应、中断与继续

---

## 1. Agent Runtime 目标

Agent Runtime 是本项目的核心。它负责把用户自然语言任务转化为可执行的多步开发流程，包括：

- 分析任务；
- 构建上下文；
- 调用模型；
- 调用工具；
- 生成计划；
- 修改代码；
- 请求权限；
- 运行验证；
- 输出总结。

---

## 2. 对话输入

### 功能编号

F7

用户可以在聊天框中输入自然语言任务。

支持：

- 普通问答；
- 针对当前项目提问；
- 请求修改代码；
- 请求运行测试；
- 请求解释文件；
- 请求生成文档。

---

## 3. Agent 任务模式

### 功能编号

F8

对于涉及代码修改或命令执行的请求，Agent 应进入任务模式。

任务模式流程：

```text
用户输入任务
↓
Agent 判断任务类型
↓
读取项目上下文
↓
生成任务计划
↓
用户确认计划，可选
↓
执行工具调用
↓
生成或修改代码
↓
展示 diff
↓
用户确认应用
↓
运行验证命令
↓
输出最终总结
```

---

## 4. Agent Loop

推荐 Agent Loop：

```text
while task not completed:
  1. 构建当前上下文
  2. 调用模型生成下一步意图
  3. 如果需要工具，执行工具
  4. 收集工具结果
  5. 更新任务状态
  6. 如果需要用户权限，暂停等待
  7. 如果产生 diff，等待用户确认
  8. 如果需要验证，运行验证命令
  9. 根据结果继续或结束
```

Agent Loop 必须有上限：

- 最大工具调用次数；
- 最大修复循环次数；
- 最大模型请求次数；
- 最大执行时间。

避免 Agent 无限运行。

## 5. 任务计划展示

### 功能编号

F9

Agent 在执行复杂任务前应展示计划。

计划内容包括：

- 目标；
- 影响范围；
- 涉及文件；
- 执行步骤；
- 风险点；
- 验证方式。

### 计划确认策略

| 任务类型 | 是否需要计划确认 |
|---|---|
| 普通问答 | 不需要 |
| 只读项目分析 | 可选 |
| 修改单文件 | 建议需要 |
| 修改多文件 | 必须需要 |
| 执行命令 | 必须需要 |
| 删除文件 / Git push / 发布 | 强确认或默认禁止 |

---

## 6. 流式响应

### 功能编号

F10

Agent 回复应支持流式输出，提升体验。

要求：

- 模型输出边生成边展示；
- 工具调用时显示正在执行的工具；
- 工具结果返回后继续生成；
- 允许用户中断生成；
- 流式事件应能区分文本、工具调用、工具结果、错误和完成。

---

## 7. 中断与继续

### 功能编号

F11

用户可以中断当前任务，也可以继续上一次任务。

要求：

- 中断后保留已完成步骤；
- 未应用的 diff 不应自动写入；
- 继续时能恢复任务计划、上下文和日志；
- 如果工作区文件已发生变化，应提示用户重新确认；
- 中断命令执行时，需要尽量终止子进程。
