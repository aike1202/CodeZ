# 11. UI/UX 需求

> 模块：主界面布局、页面模块、交互原则

---

## 1. 主界面布局

推荐布局：

```text
┌─────────────────────────────────────────────────────────────┐
│ 顶部栏：项目名 / 模型选择 / 设置 / 运行状态                   │
├───────────────┬─────────────────────────┬───────────────────┤
│ 左侧文件树     │ 中间 Agent Chat          │ 右侧上下文/diff     │
│               │                         │                   │
│ - 文件列表     │ - 用户消息               │ - 任务计划          │
│ - 搜索         │ - Agent 回复             │ - 涉及文件          │
│ - Git 状态     │ - 工具调用过程           │ - diff 预览         │
│               │ - 输入框                 │                   │
├───────────────┴─────────────────────────┴───────────────────┤
│ 底部面板：终端输出 / 测试结果 / 日志                          │
└─────────────────────────────────────────────────────────────┘
```

---

## 2. 页面模块

必须包含：

1. 欢迎页；
2. 打开项目页；
3. 主工作区；
4. 模型设置页；
5. Agent 对话页；
6. diff 审查页；
7. 命令输出面板；
8. 任务历史页；
9. 应用设置页。

---

## 3. 交互原则

1. 用户始终知道 Agent 正在做什么；
2. 高风险操作前必须确认；
3. 所有代码变更必须可审查；
4. 错误信息必须可读；
5. 长任务必须可中断；
6. 任务完成后必须有总结；
7. 默认保护用户代码，不做无提示破坏性操作。

---

## 4. 关键 UI 组件

### 4.1 Chat Panel

用于展示：

- 用户消息；
- Agent 消息；
- 流式回复；
- 工具调用过程；
- 错误提示；
- 输入框。

### 4.2 File Tree

用于展示：

- 当前 Workspace 文件；
- 文件搜索；
- Git 状态，可选；
- 被修改文件高亮。

### 4.3 Diff Viewer

用于展示：

- 文件修改前后对比；
- 接受/拒绝按钮；
- 单文件操作；
- 修改原因。

### 4.4 Terminal Output Panel

用于展示：

- 命令内容；
- stdout；
- stderr；
- 退出码；
- 运行耗时；
- 终止按钮。

### 4.5 Permission Dialog

用于展示：

- Agent 想做什么；
- 为什么要做；
- 风险等级；
- 允许/拒绝/本任务内允许。

---

## 5. 可访问性和体验

- 支持深色主题；
- 支持快捷键，可选；
- 长文本区域支持复制；
- 错误信息需要可读，不只显示 raw error；
- 加载状态明确；
- 长任务显示进度。
