# 01. 产品概述

> 模块：产品目标、业务背景、目标用户、产品边界、MVP 范围

---

## 1. 一句话描述

开发一款基于大语言模型 API 的桌面端 Agent Coding 工具，使用户可以在本地打开代码项目，通过自然语言让 Agent 分析项目、规划任务、修改代码、运行命令、验证结果，并以可审查、可确认、可回滚的方式完成软件开发任务。

---

## 2. 项目背景

当前主流 AI 编程工具正在从“代码补全”向“Agent Coding”演进。传统代码补全工具只能辅助编写局部代码，而 Agent Coding 工具可以围绕一个完整开发任务执行多步操作，例如：

- 阅读项目结构；
- 搜索相关代码；
- 理解框架和业务逻辑；
- 制定实现计划；
- 修改多个文件；
- 运行测试和构建命令；
- 根据错误日志自动修复；
- 向用户展示 diff 并等待确认；
- 记录任务过程和项目记忆。

本项目希望实现一个桌面端 Agent Coding 工具，作为学校项目和长期产品原型。第一阶段不训练模型，而是通过统一 API 接入不同大模型厂商，重点建设 Agent Runtime、工具调用、上下文管理、桌面交互和安全权限系统。

---

## 3. 项目目标

本项目的目标不是重新训练大模型，而是构建一个稳定、可扩展、可验证的编程 Agent 桌面应用。核心目标包括：

1. 支持桌面端打开本地代码项目；
2. 支持通过自然语言提交编程任务；
3. 支持接入多个模型厂商 API；
4. 支持 Agent 自动读取、搜索和理解项目；
5. 支持 Agent 制定任务计划并逐步执行；
6. 支持安全地修改代码文件；
7. 支持运行终端命令、测试命令和构建命令；
8. 支持展示文件变更 diff，让用户确认是否应用；
9. 支持任务过程记录、项目记忆和后续继续执行；
10. 支持分阶段迭代，每个阶段都有可运行测试和验收标准。

---

## 4. 目标用户

| 用户类型 | 使用场景 | 主要诉求 |
|---|---|---|
| 学生开发者 | 课程设计、毕业设计、个人项目 | 快速完成项目开发、理解代码、生成文档 |
| 独立开发者 | 本地项目开发、Bug 修复、功能迭代 | 提升开发效率，减少重复劳动 |
| 初级程序员 | 学习框架、阅读项目、辅助编码 | 通过 Agent 理解工程结构和代码逻辑 |
| 高级开发者 | 重构、测试补全、批量修改 | 可控、可审查、可验证地让 Agent 修改项目 |

---

## 5. 产品边界

### 5.1 当前阶段包含

- 桌面端应用；
- 本地项目打开与文件读取；
- 多模型 API 配置；
- Agent 对话；
- 文件搜索；
- 文件修改；
- diff 展示；
- 命令执行；
- 测试验证；
- 任务状态记录；
- 基础项目记忆；
- 权限确认。

### 5.2 当前阶段不包含

- 自训练大模型；
- 复杂云端协同编辑；
- 多人实时协作；
- 完整 IDE 替代能力；
- 云端代码托管；
- 自动发布生产环境；
- 无确认地执行高风险命令；
- 绕过系统权限或安全策略。

---

## 6. MVP 范围

### 6.1 MVP 必须包含

- Electron + React 桌面应用；
- 打开本地项目；
- 文件树；
- 配置 OpenAI-compatible API；
- 基础聊天；
- 只读工具调用；
- 任务计划；
- 文件修改 diff；
- 用户确认后应用变更。

### 6.2 MVP 可以暂缓

- 多 Provider 深度适配；
- 自动修复循环；
- 复杂项目记忆；
- Monaco 高级编辑器能力；
- 跨平台安装包；
- 语义向量检索；
- LSP 集成；
- Tree-sitter 代码结构索引。
