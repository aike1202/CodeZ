# 08. 上下文管理与项目记忆

> 模块：项目摘要、上下文选择、上下文预算、项目记忆

---

## 1. 项目摘要生成

### 功能编号

F19

系统需要在打开项目后生成基础项目摘要。

摘要包括：

- 项目名称；
- 技术栈；
- 入口文件；
- 脚本命令；
- 主要目录；
- 测试方式；
- 构建方式。

---

## 2. 上下文选择

### 功能编号

F20

Agent 不应将整个项目全部发送给模型，而应选择相关上下文。

选择依据：

- 用户任务关键词；
- 当前打开文件；
- 搜索结果；
- 项目摘要；
- 最近修改文件；
- 错误日志；
- package scripts。

---

## 3. 上下文预算控制

### 功能编号

F21

系统需要限制发送给模型的内容大小。

要求：

- 按模型上下文长度设置 token 预算；
- 优先保留高相关文件；
- 大文件只发送片段；
- 对低相关内容做摘要；
- 超出预算时提示上下文截断；
- 不同模型可配置不同预算。

---

## 4. 上下文构建流程

推荐流程：

```text
用户任务
↓
提取关键词和意图
↓
读取项目摘要
↓
搜索相关文件
↓
读取高相关文件片段
↓
加入当前打开文件和错误日志
↓
按 token 预算排序与裁剪
↓
生成最终模型上下文
```

---

## 5. 项目记忆

### 功能编号

F22

系统应保存项目级记忆。

内容包括：

- 技术栈；
- 编码规范；
- 常用命令；
- 项目结构说明；
- 用户偏好；
- 已知问题；
- 最近任务总结。

存储位置建议：

```text
.agent/
├── project-memory.md
├── tasks/
└── index.json
```

也可以存储到应用全局 SQLite 中。

---

## 6. 项目记忆更新策略

项目记忆不应由模型随意覆盖。建议采用：

- 追加为主；
- 重要变更需要用户确认；
- 记录更新时间；
- 支持查看历史；
- 支持清除记忆。

---

## 7. 敏感上下文策略

默认不进入上下文的文件：

- `.env`；
- `.env.*`；
- `*.pem`；
- `*.key`；
- `id_rsa`；
- `id_ed25519`；
- `credentials.json`；
- `secrets.*`。

如果用户明确要求读取敏感文件，系统必须弹窗说明风险并请求确认。
