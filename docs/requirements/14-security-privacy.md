# 14. 权限、安全与隐私策略

> 模块：操作风险分级、Workspace 边界、敏感文件、隐私日志

---

## 1. 操作风险分级

| 风险等级 | 操作 | 默认策略 |
|---|---|---|
| 低 | 读取普通项目文件 | 可自动执行，记录日志 |
| 中 | 写入新文件、修改文件 | 展示 diff，用户确认 |
| 中 | 运行测试、构建命令 | 展示命令，用户确认 |
| 高 | 删除文件、安装依赖 | 二次确认 |
| 高 | Git commit | 用户确认 |
| 极高 | Git push、发布、执行远程脚本 | 默认禁止或强确认 |

---

## 2. Workspace 边界

所有文件路径必须经过 normalize 和校验，确保不能通过 `../` 等方式访问 Workspace 外部文件。

要求：

- 所有文件工具必须传入 workspaceRoot；
- 所有相对路径解析后必须仍在 workspaceRoot 内；
- 符号链接需要谨慎处理；
- 命令执行默认工作目录必须是 workspaceRoot；
- 禁止工具默认访问用户主目录、系统目录或其他磁盘路径。

---

## 3. 敏感文件策略

默认敏感文件：

- `.env`；
- `.env.*`；
- `*.pem`；
- `*.key`；
- `id_rsa`；
- `id_ed25519`；
- `credentials.json`；
- `secrets.*`。

策略：

- 默认不读取；
- 默认不发送给模型；
- 用户主动确认后才可读取；
- 日志中必须脱敏。

---

## 4. 隐私日志控制

日志中不能默认明文记录：

- API Key；
- 完整敏感代码，可配置；
- 用户密码；
- token；
- `.env` 内容；
- 私钥内容。

---

## 5. 模型请求隐私提示

首次配置远程模型或首次发送项目上下文前，应提示用户：

- 模型请求可能发送部分代码片段；
- 发送范围由上下文管理模块控制；
- 敏感文件默认排除；
- 用户可以关闭远程模型或使用本地模型；
- 用户可以清除会话和项目记忆。

---

## 6. 高风险命令策略

高风险命令包括：

- 删除大量文件；
- 修改系统配置；
- 安装全局依赖；
- 执行远程脚本；
- Git push；
- 发布 npm 包或部署生产环境。

默认策略：

- 阻止或强确认；
- 展示命令原文；
- 展示风险说明；
- 不允许 Agent 静默执行。
