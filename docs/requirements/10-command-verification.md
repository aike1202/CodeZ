# 10. 命令执行、测试验证与修复循环

> 模块：内置命令输出、验证命令识别、测试失败修复循环

---

## 1. 内置命令输出面板

### 功能编号

F26

系统需要展示 Agent 执行命令的输出。

要求：

- 展示命令；
- 展示工作目录；
- 展示 stdout；
- 展示 stderr；
- 展示退出码；
- 展示耗时；
- 支持复制输出；
- 支持终止正在运行的命令。

---

## 2. 自动识别验证命令

### 功能编号

F27

系统应根据项目类型识别推荐命令。

### Node.js 项目

- `npm test`；
- `npm run test`；
- `npm run lint`；
- `npm run build`；
- `pnpm test`；
- `pnpm build`。

### Python 项目

- `pytest`；
- `python -m pytest`；
- `python main.py`。

### Java 项目

- `mvn test`；
- `gradle test`。

### Go 项目

- `go test ./...`。

### Rust 项目

- `cargo test`。

---

## 3. 命令执行权限

执行命令前必须展示：

- 命令内容；
- 工作目录；
- 执行目的；
- 风险等级；
- 是否会修改文件；
- 是否可能访问网络。

用户确认后才能执行。

---

## 4. 测试失败修复循环

### 功能编号

F28

如果测试失败，Agent 可以读取错误日志并尝试修复。

要求：

- 最多自动循环次数可配置；
- 每次循环都记录变更原因；
- 超过次数后停止并总结；
- 不允许无限循环；
- 每次修复仍然需要经过 diff 审查；
- 每次重新执行命令仍然需要遵守权限策略，除非用户授予本任务内自动重跑权限。

---

## 5. 验证总结

每次验证结束后，Agent 应输出总结：

- 运行了哪些命令；
- 每个命令是否成功；
- 失败命令的关键错误；
- Agent 做了哪些修复；
- 是否仍有未解决问题；
- 下一步建议。

---

## 6. 验收标准

- Agent 执行命令前显示命令、工作目录和目的；
- 用户确认后命令才运行；
- 命令输出实时显示；
- 命令完成后显示退出码；
- 命令超时可终止；
- 测试失败时 Agent 能读取错误并提出修复建议；
- 修复循环次数可配置；
- 最终总结包含运行了哪些命令、结果如何。
