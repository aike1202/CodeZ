## 整体技术与架构总览
本阶段的主轴是 **上下文与历史状态的持久化**。应用由“一次性工具”转变为“具备长期记忆的智能助手”。
核心技术实现包含三个层级：
1. **持久层（Main Process）**：基于 JSON 文件实现各类数据的读写缓存库 (`SessionStore`, `TaskStore`, `ProjectMemoryStore`, `RecentProjectsStore`)。
2. **通讯层（IPC）**：桥接渲染进程的操作，以安全可控的方式读取与写入文件。
3. **展示与逻辑层（Renderer Process）**：结合 Zustand store 或 React Context 同步主进程数据，实现界面上的会话列表、最近项目列表及任务查询，并将这些记忆拼接到大模型提示词中。

## 阶段与任务大纲

**目标**：为应用引入会话管理、项目长期记忆和任务记录留存机制，并在 UI 上呈现这些功能入口。

⏳ 第一阶段 · 持久层与服务扩展
  ✅ 1、[任务 T1] 扩展 `SessionStore` 及其 IPC
     - 极简说明：在 `SessionData` 中增加 `isArchived?: boolean` 字段；实现对应归档和删除的业务方法及 IPC 处理器。
  ✅ 2、[任务 T2] 新增 `ProjectMemoryStore` 和 `TaskStore`
     - 极简说明：在 `src/main/services/` 中实现两个用于持久化项目结构记忆（ProjectMemory）和执行日志（TaskData）的仓库，并注册相应的 IPC handle。
  ✅ 3、[任务 T3] 完善 `RecentProjectsStore` 集成
     - 极简说明：目前服务已存在，需要补充对应的 IPC 通道（如 `WORKSPACE_GET_RECENT`）并在主进程注册。

⏳ 第二阶段 · 前端逻辑与 UI 整合
  ✅ 1、[任务 T4] 开发“会话管理” UI
     - 极简说明：在左侧或右侧 Sidebar 新增区域，展示“活跃会话”与“已归档会话”。支持新建对话、切换历史、点击归档和删除。
  ✅ 2、[任务 T5] 开发“最近打开项目”与欢迎页交互
     - 极简说明：当未加载项目时，展示由 `RecentProjectsStore` 返回的项目列表，支持点击一键重载项目。
  ✅ 3、[任务 T6] 整合 Project Memory 和 Task History 入口
     - 极简说明：为项目记忆（Project Memory）增加管理或清除入口，在提交大模型请求时将记忆作为系统提示拼接进去；提供查看任务历史的悬浮窗或独立面板。

### 验收&测试点
  ✅ 1、[会话切换与归档]：创建多个对话、切换并发送消息。点击归档后查看归档列表。重启应用后会话完全恢复。
  ✅ 2、[最近项目加载]：打开新项目后重启，应用在欢迎页出现该项目的快速入口。点击后顺利载入树状结构。
  ✅ 3、[项目记忆注入]：打开复杂项目后让大模型记住某几个偏好，后续的新会话中，大模型依然能复述并应用该偏好。
