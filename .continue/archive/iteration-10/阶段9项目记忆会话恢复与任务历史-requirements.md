## 需求概述

**一句话描述**：实现项目的本地存储能力，让应用可以记住项目背景、用户偏好、会话历史，并支持关闭应用后状态恢复。

**业务背景**：
当前我们的桌面 Agent 编码工具在每次重新启动后，所有的聊天会话、打开的项目以及已经完成的任务记录都会丢失。随着能力的增强，用户需要能够持续地、跨天进行项目开发，必须有持久化的上下文来协助记忆和回溯。

**预期价值**：
1. 防止上下文丢失，提升持续开发体验。
2. 避免用户每次打开工具都需要重新进行项目配置或重新描述业务背景。
3. 形成完整的“需求-开发-测试-闭环”的历史记录可供审查。

## 功能需求

### 核心功能（必须实现）

- [ ] **F1**: **会话存储与恢复 (Session Storage)**
  - 处理：基于 SQLite 或 JSON 文件，将当前进行的对话消息序列化并写入本地磁盘。应用启动时自动读取上次的最近会话并加载到聊天界面中。
  - 输出：应用重启后能看到历史对话记录。

- [ ] **F2**: **会话管理 (Session Management)**
  - 处理：支持创建新会话、删除已有会话、将会话归档（移动至单独的归档列表以便历史查阅）。
  - 输出：在侧边栏或专门的会话管理界面提供对应操作入口。

- [ ] **F3**: **项目记忆 (Project Memory)**
  - 处理：为每个选定的工作区 (Workspace) 存储关联的项目状态，包括项目架构概览、关键技术栈、用户的开发偏好等，存储在本地的全局缓存中。
  - 输出：在切换项目目录时加载相应的项目记忆。

- [ ] **F4**: **任务状态保存与历史记录 (Task Storage & History)**
  - 处理：将 Agent 生成的“计划 (Plan)”以及修改文件的差异 (Diff)、执行命令的结果与退出码关联保存为独立的任务记录。
  - 输出：应用支持查阅某个项目下的历史任务记录。

- [ ] **F5**: **最近打开项目 (Recent Workspaces)**
  - 处理：保存用户选择的根目录路径列表，并在侧边栏或欢迎页提供快速入口。
  - 输出：用户重启后点击列表项可快速重载指定工作区。

### 扩展功能（可选实现）

- [ ] **E1**: **清理缓存功能**
  - 处理：在设置界面提供“清理当前项目记忆”、“清空所有历史对话”按钮。

## 非功能需求

### 性能要求
- 数据库或文件读写速度在应用启动时不得阻塞主线程过长（保持冷启动 < 1秒）。

### 数据安全与隐私
- 相关记忆数据仅保存在本地应用配置目录中（例如 `%APPDATA%`），绝对不能上传云端。

## 验收标准

### 功能验收
- [ ] **AC1**: 打开一个新项目并发起两次对话，关闭并重启应用后，上次打开的项目及两次对话必须完好无损地呈现在 UI 上。
- [ ] **AC2**: 在一个项目下产生任务计划并点击执行，然后可以在“历史任务”或类似面板中查询到该次计划的核心步骤。
- [ ] **AC3**: 点击“清除记忆”按钮后，重新刷新应用，不再显示历史会话。
