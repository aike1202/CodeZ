## 需求概述

**一句话描述**：将现有的“项目记忆”功能从全局 JSON 及模态框配置，重构为基于项目本地的 Markdown（默认存在 `.agent/project-memory.md` 或项目根目录）文件编辑，并将功能按钮迁移至聊天输入框上方。

**业务背景**：现有的项目记忆功能必须手动通过表单维护，不够灵活，无法随代码库（Git）统一流转；并且入口按钮位置不够直观。用户期待将其统一放置在输入框工具栏，且通过直接编辑 Markdown 文件来维护。

**预期价值**：提升规则维护的便利性；使项目记忆可以跟随 Git 仓库无缝共享；UI 更符合主流 AI 工具的操作习惯；为后续更多上下文控制功能（工具栏扩展）奠定布局基础。

## 功能需求

### 核心功能（必须实现）

- [ ] **F1**: 废弃旧版 JSON 存储与 UI 模态框
  - 输入：应用初始化
  - 处理：移除 `ProjectMemoryModal` 组件、相关的 `ProjectMemoryStore` 和后端 IPC 通道。
  - 输出：清理冗余代码和无用 API。
  
- [ ] **F2**: 新增 PromptArea 快捷工具栏
  - 输入：聊天区域渲染时
  - 处理：在 `PromptArea` 的文本输入框正上方，添加一个工具栏，把“项目记忆”按钮放在首位。此工具栏设计成可横向扩展以容纳后续的新增按钮。
  - 输出：新的前端布局，移除 `TopBar` 中原有的项目记忆按钮。

- [ ] **F3**: Markdown 记忆文件自动生成与读取
  - 输入：点击新增的“项目记忆”按钮
  - 处理：调用系统 IPC 检查当前工作空间下是否存在 `.agent/project-memory.md`（或 `project-memory.md` 根据后续确认决定）。若不存在，则使用默认 Markdown 骨架创建文件。
  - 输出：将该 Markdown 文件路径作为预览路径传入，使用系统现有的 `FilePreviewPanel` 在界面右侧打开它，供用户像普通文件一样编辑。

- [ ] **F4**: 动态拼装 System Prompt 逻辑改造
  - 输入：用户发送流式对话消息时
  - 处理：替换掉旧的向主进程获取 JSON 的逻辑。从文件系统异步读取项目记忆 Markdown 文件的全文内容。
  - 输出：若有内容，直接无缝拼接到送给 LLM 的 System Message 的末尾，指导大模型的编码工作。

## 非功能需求

### 兼容性要求
- 文件系统：兼容多操作系统的路径。
- 文本编码：以 UTF-8 读取和保存项目记忆文档。

## 约束条件

### 技术栈限制
- 前端保留已有的 React 组件生态；
- 编辑器重用系统现有的右侧预览/编辑面板。

### 其他约束
- 系统将不再兼容存放在旧版 `project-memory.json` 的项目记忆。旧数据将被忽略或需用户手动迁移。

## 验收标准

### 功能验收
- [ ] **AC1**: `TopBar` 无“项目记忆”按钮，`PromptArea` 文本框上方排布着“项目记忆”按钮。
- [ ] **AC2**: 第一次点击该按钮能成功在项目生成 Markdown 骨架文档，并在右侧正确弹出预览编辑器。
- [ ] **AC3**: 发送聊天对话后，LLM 的行为受到 Markdown 文件中设定的约束规则影响。
