## 需求概述

在阶段 0-2 完成了基础工程搭建、工作区文件树（文件读取系统）管理以及初步的大模型 API 聊天打通基础之上。阶段 3 的目标是赋予应用真正的“智能体（Agent）”能力。Agent 需要能够在后台自主地基于工作区，调用预定义好的系统工具与环境进行交互。
本阶段专注于“只读（read-only）”类工具。主要包含构建基础的 Agent Loop 控制流，工具的注册与派发机制，并通过让 Agent 自动阅读、检索本地代码仓库的项目上下文，实现精准回复用户的问题，暂不涉及执行写操作或终端命令。

## 已完成功能分析 (阶段 0, 1, 2)
通过分析现有代码和项目进度(`src/main/services`, `src/renderer/src` 以及测试运行状态)，确认以下能力已经就绪：
- **阶段 0**：Electron/React/Vite 基础框架、测试与 IPC 通信。
- **阶段 1**：`WorkspaceService`、工作区建立，可以读取并展示文件树（过滤 node_modules 机制等），可以在左侧文件树点击查看代码内容，支持近排项目显示（`RecentProjectsStore.ts`）。
- **阶段 2**：支持 OpenAI 兼容的多模型厂商配置 (`ProviderService.ts`) 和流式回复聊天系统 (`ChatService.ts` / `PromptArea` 等聊天组件)。

以上功能已构成良好的基础输入、输出和操作沙盒环境。我们只需在现有的 `ChatService` / `main` 进程之上建立 `Agent` 调用闭环即可。

## 功能需求

### 1. 核心 Agent Loop 运行时 (AgentRunner)
- [ ] 构建负责管理 Agent 对话轮次的类（例如 `AgentRunner`）。
- [ ] 截获 ChatService 在调用大模型时，支持 `tools` 字段的注入以及 `tool_calls` 的解析和响应合并。
- [ ] Agent 能够在一个用户请求内进行多次的大模型循环交互（响应-调工具-提交工具结果-再产生最终响应）。

### 2. 工具注册架构 (ToolSystem/ToolManager)
- [ ] 提供标准化的工具注册机制，以便支持未来扩展阶段增加（写入、MCP 等拓展）。
- [ ] 给大模型定义统一的 Schema （OpenAI / Anthropic standard function call schema）。
- [ ] 工具调用执行调度中心，能按照大模型返回的工具名和参数寻找对应的函数实现，返回 JSON 运行结果。

### 3. 三个核心只读工具实现
- [ ] **list_files 工具**：接受一个路径作为参数，读取工作区对应的目录下的一级或多级文件列表结构（类似 `ls` 或递归的 tree）。
- [ ] **read_file 工具**：接受绝对或相对路径作为参数，返回限定数量的源码文件内容。
  - 需要防御性保护（限制读取不可跨出 Workspace `../` 目录区域）。
  - 超大文件读取时需截断或限制范围（避免污染 Token context）。
- [ ] **search_text 工具**：接受正则表达式或关键字以及目录参数，执行全文或者代码检索，返回匹配行以及相关文件路径（类似 `grep` 或 `ripgrep` 功能）。

### 4. 工具调用的前端感知与展示 (UI)
- [ ] 后端（主进程）在执行某工具调用开始和结束时发布进度。
- [ ] 前端对话框需能够渲染在流式生成答案前，展示一个“工具调用过程块”，如 “正在读取 /src/App.tsx... 成功” （Tool Call Logs）。
- [ ] 确保只读工具可以自动被 Agent 执行而**不需**像写入工具那样总是弹出强制确认框，平滑阅读体验。

## 非功能需求
- **安全性**：读取文件的范围需严格控制在当前 Workspace 的指定目录下。
- **性能/Token 预算**：因为 Agent 可以自发读取文件，模型一次索取大量文件时应当有对应的长度拦截并提示“超出长度上限”，而不是完全无视上限传递。
- **可维护性**：`AgentRunner` 以及每个功能的 `Tool` 务必高度解耦，以便后期注入阶段 6、9 的命令与 MCP 测试功能。

## 约束条件
- 框架与包：本阶段不需要使用第三方复杂的智能体框架，利用原生的 OpenAI 兼容格式手动维护 messages context 以及解析工具流即可。
- 大模型兼容性：保证主要使用的 `ollama`，`openai`, `anthropic` 厂商等格式对齐。

## 验收标准
- [ ] 开发完成并集成 `list_files`, `read_file`, `search_text` 工具到后端的 Agent 工具包列表中。
- [ ] 当给 Agent 输入“请告诉我本项目的 package.json 中的 npm scripts 有哪些”，Agent 能够自主通过只读工具读取 `package.json`，并结合文件内容作出正确回复。
- [ ] 当告诉 agent，“帮我通过搜素查询项目中在哪里定义了 WorkspaceService 类的实例”，他可以自主调阅 search 工具找到目标进行解释。
- [ ] Agent 工具执行的过程状态会实时输出显示在前端界面的聊天消息之上。
- [ ] Agent 尝试读取当前工作区目录以外的文件（如 `/etc/passwd` 或 `C:\Windows\...`）被严格阻止并返回错误信息进入错误上下文。
