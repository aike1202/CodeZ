# 阶段 8：体验优化、打包发布与跨平台准备

## 背景
根据项目路线图，目前已经完成核心的 Agent Loop、文件修改和终端执行功能。下一步需要将应用打磨为可独立运行的桌面级软件。

## 目标
完善 UI、错误处理、安全提示和应用打包能力，形成可演示、可安装的桌面软件。

## 需求范围
1. **完善主界面视觉与深色主题**：应用全局支持深色模式（根据系统主题自动切换或手动切换），优化各组件的暗黑模式配色。
2. **优化错误提示与完善日志查看**：处理全局异常，并优化前端/主进程日志展示。
3. **增加全局快捷键（可选）**：通过快捷键快速唤起或隐藏主窗口。
4. **配置应用图标与 electron-builder**：引入并配置 `electron-builder` 以支持多平台打包，配置项目 icon。
5. **打包 Windows 安装包**：能够顺利构建出 `.exe` 安装程序。
6. **跨平台准备**：在 `electron-builder` 配置中加入对 macOS/Linux 的打包支持选项。

## 约束条件
- 不破坏现有核心对话流程和命令执行模块。
- 遵循 Electron 安全最佳实践。
