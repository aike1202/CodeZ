# 🏗️ 项目背景 - MyAgent

> 创建时间：2026-06-27 14:30
> 最后更新：2026-06-27 14:30
> 全局状态索引：见 .continue/index.md

## 项目目标
开发一个基于 Electron 的本地 AI 智能体编码桌面客户端，通过跨进程 IPC 及强大的组件化，实现项目上下文的高效管理与自然对话。

## 业务背景
本地开发者需要能够快速切换上下文、管理多项目工作流、并约束 AI 开发规范的桌面助手。痛点在于多项目上下文切换时常常忘记技术栈与架构规则。

## 系统范围
桌面端应用程序（Electron），包括主进程（负责文件 I/O、配置读取、Terminal 管理等）和渲染进程（UI 组件，负责对话、项目导航、配置填写等）。

## 整体架构
基于 Electron 构建。
主进程：Node.js 驱动，管理配置文件与系统级操作，IPC 暴露 API。
渲染进程：React 视图层，处理交互逻辑和与大模型的对话拼接。

## 技术栈
| 层 | 选型 |
|----|------|
| 桌面框架 | Electron |
| 前端框架 | React, TypeScript, Vite |
| 样式 | Tailwind CSS |
| 状态管理 | Zustand |
| 数据存储 | 纯文件 (JSON / Markdown) |

## 约束条件
- 严格遵循 TypeScript 类型规范。
- 禁止 UI 侧直接执行 I/O 或系统命令，需通过 IPC 与 `preload` 桥接层交互。
- 遵循统一的项目记忆约束管理策略。
- （v2第一轮优化新增）**权限约束**：工具系统必须过统一 PermissionManager，写操作默认 ask/allow。
- （v2第一轮优化新增）**防呆机制**：长对话或压缩上下文时，需要有结构化的 ResumeState 防丢失。
